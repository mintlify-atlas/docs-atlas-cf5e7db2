## Description

Tests TCP/TLS connection, authenticates with the IMAP server, and retrieves the server's capability list. Use this tool to diagnose connection issues or verify account configuration.

<Info>
This tool executes a full connection test including NOOP command to verify bidirectional communication.
</Info>

## Input Parameters

<ParamField path="body.account_id" type="string" default="default">
  Account identifier matching an environment variable configuration.
  
  Pattern: `^[A-Za-z0-9_-]{1,64}$`
</ParamField>

## Response

<ResponseField name="summary" type="string" required>
  Human-readable summary of the verification outcome.
  
  Example: `"Account verification succeeded"`
</ResponseField>

    <ResponseField name="issues" type="object[]" required>
      Array of diagnostic issues encountered during verification.

        <ResponseField name="code" type="string" required>
          Error code: `"invalid_input"`, `"auth_failed"`, `"not_found"`, `"timeout"`, `"conflict"`, or `"internal"`.
        </ResponseField>
        
        <ResponseField name="stage" type="string" required>
          Operation stage where issue occurred (e.g., `"connect_authenticated"`, `"noop"`, `"capabilities"`).
        </ResponseField>
        
        <ResponseField name="message" type="string" required>
          Actionable error message.
        </ResponseField>
        
        <ResponseField name="retryable" type="boolean" required>
          Whether the operation can be retried.
        </ResponseField>
      </Expandable>
    </ResponseField>
    
    <ResponseField name="next_action" type="object" required>
      Suggested follow-up action.
      
      <Expandable title="properties">
        <ResponseField name="instruction" type="string" required>
          Human-readable instruction for next step.
        </ResponseField>
        
        <ResponseField name="tool" type="string" required>
          Recommended tool to call next.
        </ResponseField>
        
        <ResponseField name="arguments" type="object" required>
          Suggested arguments for the next tool call.
        </ResponseField>
      </Expandable>
    </ResponseField>
    
    <ResponseField name="account_id" type="string" required>
      Account identifier that was verified.
    </ResponseField>
    
    <ResponseField name="ok" type="boolean" required>
      Whether verification succeeded (`true` unless `status="failed"`).
    </ResponseField>
    
    <ResponseField name="latency_ms" type="number" required>
      Round-trip latency to IMAP server in milliseconds.
    </ResponseField>
    
    <ResponseField name="server" type="object" required>
      Server connection details.
      
      <Expandable title="properties">
        <ResponseField name="host" type="string" required>
          IMAP server hostname.
        </ResponseField>

      IMAP server capabilities (max 256). Empty array if capabilities fetch failed.
      
      Common capabilities include: `"IMAP4rev1"`, `"IDLE"`, `"MOVE"`, `"UIDPLUS"`, `"STARTTLS"`.
    </ResponseField>

<ResponseField name="meta" type="object" required>

## Timeout Configuration

Connection timeouts are configurable via environment variables:

- `MAIL_IMAP_CONNECT_TIMEOUT_MS` (default: 30000) - TCP/TLS connection timeout
- `MAIL_IMAP_GREETING_TIMEOUT_MS` (default: 15000) - Server greeting timeout
- `MAIL_IMAP_SOCKET_TIMEOUT_MS` (default: 300000) - Socket read/write timeout
